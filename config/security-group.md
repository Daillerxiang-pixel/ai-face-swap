# 在对应云控制台（如阿里云 ECS、DigitalOcean Firewall 等）配置入站规则

## 入方向规则

| 端口 | 协议 | 来源 | 用途 | 优先级 |
|------|------|------|------|--------|
| 22 | TCP | 0.0.0.0/0 或你的 IP | SSH 远程登录 | 高 |
| 80 | TCP | 0.0.0.0/0 | HTTP（Nginx） | 高 |
| 443 | TCP | 0.0.0.0/0 | HTTPS（Nginx） | 高 |

## ⚠️ 不需要开放的端口

| 端口 | 说明 |
|------|------|
| 8080 | Node.js 直连端口，仅 127.0.0.1 访问，Nginx 反代即可 |
| 3306 | 不使用 MySQL |
| 6379 | 不使用 Redis |

## 建议优化

1. **SSH 端口**：把来源改为你的固定 IP（如 `你的IP/32`），不要对全互联网开放
2. **如需限制 API 访问**：80/443 保持全开，后续可通过 Nginx 或 WAF 防护
